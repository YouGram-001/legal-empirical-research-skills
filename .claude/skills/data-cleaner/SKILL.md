# 裁判文书数据清洗与同案归总

排除无效样本，识别并归总关联案件，输出清洗后的索引表。

## 触发条件

- "清洗数据"、"数据清洗"、"去重"
- "排除无效样本"、"关联案件归总"
- "同案归总"、"生成事件归总ID"

## 工作流程

### Step 1: 确认清洗规则

先与用户确认（可接受默认值）：

```
默认排除规则：
  1. 完全重复的文书（案号 + 当事人 + 裁判日期完全一致）
  2. 内容严重残缺的文书（缺少"本院认为"或"裁判结果"部分）

默认归总规则：
  - 按"区/县 + 年份 + 核心纠纷类型"生成事件归总ID
  - 自动标注审级关系（一审/二审/再审/其他）

需要调整吗？
```

### Step 2: 执行清洗

调用 `scripts/clean_dedupe.py`：

```bash
python .claude/skills/data-cleaner/scripts/clean_dedupe.py \
  --index "output/index_raw.csv" \
  --raw-dir "output/raw_texts" \
  --output "output/cleaned_index.csv" \
  --excluded-dir "output/excluded"
```

脚本自动：
1. 读取 index_raw.csv，加载所有转化后的文本
2. 完整性检查：验证"本院认为"和"裁判结果"是否存在
3. 提取当事人姓名，构造去重键（案号+当事人+日期）
4. 标记完全重复的文书
5. 提取区/县（从审理法院）、年份（从案号/日期）、纠纷类型（关键词匹配）
6. 生成事件归总ID并标注审级
7. 排除文件移至 `excluded/` 文件夹

### Step 3: 汇报结果

清洗完成后汇报：
- 保留/排除数量及原因分布
- 事件归总ID数量和含多份文书的事件组
- 审级分布概况
- 下一步建议："数据已清洗完成，要开始信息抽取吗？"

## 排除标准详情

| 规则 | 检测方式 | 处理 |
|------|----------|------|
| 完全重复 | 案号 + 当事人 + 裁判日期 完全一致 | 只保留第一份，其余标记排除 |
| 内容残缺 | 缺少"本院认为"或"裁判结果"段落 | 标记排除 |
| 全文过短 | 文本长度 < 200 字符 | 标记排除 |

## 事件归总ID

格式：`{区/县}_{年份}_{核心纠纷类型}`

例如：
- `广东省深圳市_2019_集体经济组织权益`
- `浙江省杭州市_2017_行政许可`

核心纠纷类型通过关键词匹配自动分类：
- "征收"、"补偿款"、"征地" → 土地征收补偿分配
- "就业"、"录用"、"招聘" → 就业歧视
- "集体经济"、"成员资格" → 集体经济组织权益
- 等（可在 research_config.yaml 中扩展）

## 审级标注

| 标记 | 检测模式 |
|------|----------|
| 一审 | 一审、初字、简易程序 |
| 二审 | 二审、终字、终审、上诉 |
| 再审 | 再审、审判监督、再字 |
| 其他 | 无法识别 |

## 输出

- `output/cleaned_index.csv` — 清洗后索引表，包含：
  - 案件ID、原始文件名、转化后路径
  - 案号、审理法院、审结日期、确认审级、案由、当事人
  - 文本长度、完整性、缺失段落、排除原因、保留标记
  - 事件归总ID、区/县、裁判年份、核心纠纷类型
- `output/excluded/` — 被排除的文书备份（含排除原因记录）

## 注意事项

- 排除的文书移入 `excluded/` 文件夹备查，不直接删除
- 模糊去重（余弦相似度）暂未启用，如需要可在 config 中配置
- 纠纷类型分类基于关键词，边界不清的案件可能分类不准确（后续可人工修正）
- 事件归总ID 中的区/县依赖于审理法院名称，部分法院名称可能无法精确提取
