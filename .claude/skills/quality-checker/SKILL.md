# 数据质量检验

## 触发条件

- "检查数据质量"、"质量检验"、"质量报告"
- "验证抽取结果"、"有没有编造的"
- "检查幻觉"

## 工作流程

### Step 1: 缺失率分析（快速扫描）

```bash
python .claude/skills/quality-checker/scripts/quality_report.py \
  --input output/final_labeled_data.csv \
  --output output/quality_report.txt
```

快速过，不是重点。

### Step 2: 抽样幻觉检测（核心步骤）← 必须执行

#### 2.1 确定抽样量

抽样比例按以下规则自动计算（基于 `final_labeled_data.csv` 总记录数 N）：

| N 范围 | 抽样方式 |
|--------|----------|
| N < 100 | **逐一检验**（100%，全部） |
| 100 ≤ N < 1000 | **30%**（至少 20 条） |
| N ≥ 1000 | **15%**（至少 50 条） |

用 `sample_for_review.py` 执行分层抽样（按审级 + 防卫认定结果分层，确保代表性）。

#### 2.2 逐条比对原文

对每条抽样记录，读取对应的原始裁判文书纯文本（`output/raw_texts/` 中对应的 .txt），
逐字段比对 AI 抽取结果与原文：

| 字段类型 | 检测方法 |
|----------|----------|
| **客观字段**（案号/法院/日期/审级） | 正则匹配 + 原文定位 |
| **二分类标签**（主张防卫/互殴/1173条等） | 原文关键词搜索 → 判断 0/1 是否有原文依据 |
| **分类标签**（冲突起因/关系/伤情对比） | 读取原文事实段落 → 判断分类是否合理 |
| **数值字段**（赔偿额/赔偿比例） | 在原文中搜索数字 → 确认数值是否来自原文 |
| **开放文本**（事件概述/防卫要点/裁判要旨） | 逐句核对关键事实陈述 → 检查是否有原文未提及的编造内容 |

#### 2.3 幻觉分级

| 级别 | 定义 |
|------|------|
| 🔴 **编造** | 原文完全没有依据，AI 自己编的（漏提原文明确记载的信息也算编造） |
| 🟡 **偏差** | 大致正确但数值或细节有误 |
| ➖ **无法判断** | 原文信息不足，无法验证真伪 |

#### 2.4 系统性错误自动标记

如果同一字段在本次抽样中出现编造的比例超过抽样量的 **50%**，
自动将该字段标记为"系统性错误"——该字段在全量数据中都需要回到 info-extractor 重抽，无需等用户逐条确认每条记录。

示例：
```
⚠ 系统性错误: "原告主张赔偿额" — 抽样 20 条中 12 条为 null 但原文有值 (60% > 50%)
   → 建议: 全量重抽此字段，调整 prompt 增加"在原文首部诉请段落中查找原告主张总额"的指令
```

### Step 3: 用户逐案确认（交互式）

将幻觉报告中的 🔴 和 🟡 条目逐条展示给用户确认。

**不得跳过此步骤** — 只有用户能最终判断是否真的编造了。

### Step 4: 迭代直到清零 ← 强制规则

⚠️ **只要本次抽样出现任何 🔴 编造**，必须执行以下迭代：

1. **加大样本**：将抽样量翻倍（如第一轮抽 20 条 → 第二轮抽 40 条）
2. **打乱重抽**：重新随机抽样（排除已检测过的记录）
3. **再次检测**：对新样本执行同样的幻觉检测
4. **重复**：直到**连续两轮抽样均无 🔴 编造**，且每轮 🟡 偏差数不超过抽样量的 5%

```
迭代示例：
  第 1 轮: 抽 20 条 → 发现 5 处编造 → 用户确认 + 修复
  第 2 轮: 抽 40 条 → 发现 1 处编造 → 用户确认 + 修复
  第 3 轮: 抽 80 条 → 0 编造, 1 偏差 ✅
  第 4 轮: 抽 80 条 → 0 编造, 0 偏差 ✅  ← 连续两轮清零，通过
```

如果数据总量不足以继续翻倍，则检验全部剩余记录。

### Step 5: 汇总

```
质量检验完成 ✅

📊 缺失率概览: 见 quality_report.txt

🔍 幻觉检测:
  迭代 4 轮 | 累计检验 220 条
  连续 2 轮零编造 ✅

  字段              编造   偏差   最终准确率
  ─────────────────────────────────────
  案号               0     0     100%
  原告主张赔偿额      12    3      92%
  冲突起因            3     2      97%
  ...
```

## 输出

- `output/quality_report.txt` — 缺失率 + 分布概览
- `output/hallucination_report.md` — 每轮幻觉检测详细报告
- `output/quality_summary.csv` — 量化汇总（每字段每轮准确率）

## 注意事项

- **幻觉检测是强制步骤**，不得只做完缺失率就汇报"质量良好"
- **抽样必须分层**：覆盖不同审级（一审/二审/再审）和不同防卫认定结果
- **用户确认不可跳过**：AI 无法最终判断是否为编造
- **迭代不可中途终止**：必须连续两轮零编造才算通过
- **系统性错误自动标记**：同一字段 >50% 出错 → 全量回溯重抽
